# Legit.ly — Shortlink Specification

## Overview

Branded shortlinks are the mechanism by which Legit.ly tracks creator-driven traffic.
Every product on a creator's storefront has a unique shortlink that records clicks
and enables order attribution.

## URL Format

- **Production**: `go.legit.ly/{code}`
- **Fallback**: `legit.ly/r/{code}` (if subdomain not configured)
- **Code format**: 8-12 character URL-safe string generated by `nanoid`
- **Character set**: `[a-z0-9_-]` (lowercase alphanumeric with hyphens and underscores)

## Link Generation

### When a link is created
A tracking_link is created when a creator adds a product to their storefront.

### Uniqueness constraint
One tracking_link per unique tuple of: `(creator_id, product_id, offer_id)`

If a creator removes and re-adds the same product, reactivate the existing link
rather than generating a new code.

### Database record: tracking_links
```
id              UUID PRIMARY KEY
short_code      VARCHAR(16) UNIQUE NOT NULL
creator_id      UUID NOT NULL → creator_profiles.id
offer_id        UUID NOT NULL → offers.id
product_id      UUID NOT NULL → products.id
destination_url TEXT NOT NULL (Shopify product URL)
is_active       BOOLEAN DEFAULT true
click_count     INTEGER DEFAULT 0 (cached, derived from click_events)
created_at      TIMESTAMPTZ DEFAULT NOW()
updated_at      TIMESTAMPTZ DEFAULT NOW()
```

## Redirect Flow

```
Browser requests: GET go.legit.ly/{code}
    │
    ├── 1. Look up tracking_link WHERE short_code = {code}
    │       └── Not found or inactive? → Return 404 page
    │
    ├── 2. Bot detection
    │       └── Check User-Agent against known bot patterns
    │       └── If bot: record as bot click, still redirect
    │
    ├── 3. Click deduplication
    │       └── Generate fingerprint: hash(ip_hash + ua_hash + short_code + minute_bucket)
    │       └── If fingerprint exists in last 1 minute: skip recording, still redirect
    │
    ├── 4. Record click_event (async / non-blocking)
    │       └── INSERT into click_events table
    │
    ├── 5. Set attribution cookie
    │       └── Cookie name: _legit_attr
    │       └── Value: JSON { session_key, creator_id, offer_id, clicked_at }
    │       └── Expiry: 7 days
    │       └── Domain: .legit.ly (accessible to main domain)
    │       └── SameSite: Lax, Secure, HttpOnly
    │
    └── 6. Return 302 redirect to destination_url
```

### Performance Target
The entire redirect must complete in **under 100ms**. Click event recording
should be non-blocking (use fire-and-forget pattern or background job).

## Click Event Schema

### Database record: click_events
```
id              UUID PRIMARY KEY
tracking_link_id UUID NOT NULL → tracking_links.id
short_code      VARCHAR(16) NOT NULL
creator_id      UUID NOT NULL
offer_id        UUID NOT NULL
product_id      UUID NOT NULL
session_key     VARCHAR(64) NOT NULL
ip_hash         VARCHAR(64) NOT NULL
user_agent_hash VARCHAR(64) NOT NULL
referrer        TEXT
fingerprint     VARCHAR(128) NOT NULL
is_bot          BOOLEAN DEFAULT false
created_at      TIMESTAMPTZ DEFAULT NOW()
```

### Hashing
- IP: SHA-256 of raw IP + daily salt
- User-Agent: SHA-256 of raw UA string
- Fingerprint: SHA-256 of `ip_hash + ua_hash + short_code + minute_bucket`

### Session Key
- Generated as a 32-character `nanoid` for each new visitor
- Stored in the `_legit_session` cookie (30-day expiry)
- Reused across clicks if cookie exists
- Used for attribution matching when orders arrive

## Bot Detection

### Basic UA Patterns (MVP)
Flag as bot if User-Agent contains:
- `bot`, `crawler`, `spider`, `slurp`, `mediapartners`
- `facebookexternalhit`, `twitterbot`, `linkedinbot`
- `whatsapp`, `telegrambot`, `discordbot`
- Empty or missing User-Agent

Bot clicks are still recorded (for analytics) but marked `is_bot = true`
and excluded from attribution matching.

## DNS Configuration

### go.legit.ly subdomain
- Create CNAME record: `go.legit.ly` → Vercel deployment URL
- In the Next.js app, use middleware to detect `Host: go.legit.ly`
- If host is `go.legit.ly`, route `/{code}` to the redirect handler
- If host is `legit.ly`, route `/r/{code}` to the same redirect handler

### Middleware pseudo-code
```javascript
if (request.host === 'go.legit.ly') {
  // Rewrite go.legit.ly/{code} → /r/{code}
  const code = request.pathname.slice(1)
  return rewrite(`/r/${code}`)
}
```
